# Usability Test Plans and Scenarios

**Project Name:** Zenith Wellness App
**Goal:** Evaluate the ease of use, navigation flow, and visual clarity of the Zenith Wellness Interactive Prototype.

## 1. Test Objectives
- Evaluate the ease of navigation using the bottom tab bar.
- Assess the discoverability of interactive elements (Quick Actions, Profile Settings).
- Measure task completion time and error rates to identify UX friction points.

## 2. Methodology
- **Format :** Moderated Remote Testing with a Think-Aloud Protocol.
- **Tools:** Zoom/Google Meet for observation, built-in prototype console for event logging.
- **Duration:** 15-20 minutes per session.

## 3. Test Scenarios

### Scenario 1: Navigation & Discovery
**Prompt:** "You want to review your recent physical activities. Navigate to the Activity screen and locate your recent meditation session."
*Success Metric:* User correctly taps the 'Activity' tab on the bottom navigation and identifies the 'Mindful Meditation' card.

### Scenario 2: Core Interaction
**Prompt:** "You just drank a large glass of water and want to record it quickly in the app. Please log this action."
*Success Metric:* User taps the 'Water' quick action button on the Home screen.

### Scenario 3: Settings Adjustment
**Prompt:** "You prefer using apps with a light theme during the day. Find the setting to change this and turn off Dark Mode."
*Success Metric:* User navigates to the 'Profile' tab and successfully toggles the 'Dark Mode' switch.
