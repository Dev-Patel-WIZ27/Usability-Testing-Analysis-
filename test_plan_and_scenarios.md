# Usability Test Plan & Scenarios

## 1. Executive Summary
This document outlines the testing methodology, user scenarios, and criteria for the usability evaluation of the Zenith Wellness Interactive Prototype.

## 2. Test Objectives
- Evaluate the ease of navigation using the bottom tab bar.
- Assess the discoverability of interactive elements (Quick Actions, Profile Settings).
- Measure task completion time and error rates to identify UX friction points.
- Gather qualitative feedback on the visual design (glassmorphism, animations).

## 3. Methodology
- **Format:** Moderated Remote Testing
- **Tools:** Zoom/Google Meet for observation, built-in prototype console for event logging.
- **Duration:** 15-20 minutes per session.

## 4. Participant Profile
- **Target Audience:** Fitness enthusiasts and health-conscious individuals.
- **Age Range:** 20-35 years.
- **Sample Size:** 5 participants.

## 5. Task Scenarios

### Scenario 1: Review Activity
**Prompt:** "Imagine you just finished your day and want to check how long you meditated yesterday. Please navigate to the place where you can find this information."
**Success Criteria:** User taps the 'Activity' tab and locates the 'Mindful Meditation' item in the recent activities list.

### Scenario 2: Quick Logging
**Prompt:** "You just drank a large glass of water and want to record it quickly. Show me how you would do that."
**Success Criteria:** User taps the 'Water' quick action button on the Home screen and observes the toast notification.

### Scenario 3: Adjusting Preferences
**Prompt:** "You prefer using apps with a light theme during the day. Please find the setting to change this and turn off Dark Mode."
**Success Criteria:** User taps the 'Profile' tab, locates the 'Dark Mode' setting, and toggles the switch.

## 6. Post-Test Questionnaire
After completing the scenarios, users will be asked to rate the navigation, visual appeal, and overall experience on a scale of 1-5, and provide any open-ended feedback.
